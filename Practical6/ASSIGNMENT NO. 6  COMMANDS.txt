Bully Algorithm Practical – Complete Flow and Output Explanation

This practical simulates how a distributed system elects a new coordinator when the current coordinator fails.

1. Objective of the Program

The program simulates 5 processes:

P1, P2, P3, P4, P5

Each process can be:

Up (active) → running
Down (inactive) → failed

The process with the highest active process ID becomes the coordinator.

2. Data Structures Used
state[]
static boolean[] state = new boolean[5];

Stores whether each process is active.

Index	Process	Meaning
0	P1	true if active
1	P2	true if active
2	P3	true if active
3	P4	true if active
4	P5	true if active
coordinator
static int coordinator;

Stores the process number of the current coordinator.

3. Initial Setup (main())

At program start:

for (int i = 0; i < 5; i++) {
    state[i] = true;
}
coordinator = 5;

So:

P1 = up
P2 = up
P3 = up
P4 = up
P5 = up
Coordinator = P5
4. Menu Options
1. Up a process
2. Down a process
3. Send a message
4. Exit
5. Method Flow
up(int up)

Activates a process.

If already active → prints "already up".
If it was down → activates it and runs election.
Highest active process becomes coordinator.
down(int down)

Deactivates a process.

If already down → prints "already down".
Otherwise marks it inactive.
message(int mess)

Simulates a process sending a message.

If sender is down → prints "Process X is down".
If sender is the coordinator → prints "OK".
Otherwise starts an election.

Ring Algorithm Practical – Complete Flow and Output Explanation

This practical simulates the Ring Leader Election Algorithm used in distributed systems.

1. Objective of the Program

The goal is to elect a coordinator (leader) among multiple processes.

Rule:

The process with the highest process ID becomes the coordinator.

2. Basic Idea of Ring Algorithm

All processes are arranged in a logical ring.

Each process knows only the next process in the ring.

Election steps:

One process starts the election.
The election message circulates around the ring.
Every process adds or compares its ID.
After one full round, the highest ID is selected.
That process becomes the coordinator.
3. Example Ring

If the process IDs are:

2, 5, 8

The ring is:

2 → 5 → 8 → 2
4. Program Structure
class Process
class Process {
    int id;
    String state;
}

Each process stores:

id → process ID
state → "active"
Process[] proc
Process[] proc = new Process[n];

Stores all processes.

     ↓

     ↓
Display menu
     ↓
User selects Election or Quit
     ↓
If Election:
    Read initiator
    Find initiator position
    Pass message around ring
    Determine highest ID
    Announce coordinator
     ↓
Return to menu

8. Election Message Passing

The do-while loop sends the election message around the ring.

Iteration 1

Current = Process 5

Next process:

(1 + 1) % 3 = 2

Next = Process 8

Output:

Process 5 sends message to 8

Update:

maxId = max(5, 8) = 8
Iteration 2

Current = Process 8

Next:

(2 + 1) % 3 = 0

Next = Process 2

Output:

Process 8 sends message to 2

Update:

maxId = max(8, 2) = 8
Iteration 3

Current = Process 2

Next:

(0 + 1) % 3 = 1

Next = Process 5

Output:

Process 2 sends message to 5

The message returns to the initiator, so the loop ends.

13. Viva Explanation

In this practical, I entered three processes with IDs 2, 5, and 8. These processes were arranged in a logical ring. Process 5 initiated the election. The election message traveled from Process 5 to Process 8, then to Process 2, and finally back to Process 5. During this traversal, the program tracked the highest process ID, which was 8. Therefore, Process 8 was elected as the coordinator.
