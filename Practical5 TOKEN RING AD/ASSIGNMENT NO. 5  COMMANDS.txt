Practical 5: Token Ring Algorithm – Complete Theory

This practical is based on the Token Ring Algorithm, a token-based mutual exclusion algorithm used in distributed systems.

1. What Is a Distributed System?

A distributed system is a group of independent computers or processes that communicate over a network and work together as one system.

Examples
Online banking
ATM networks
Cloud services
Distributed databases

In distributed systems:

There is no shared memory.
There is no global clock.
Processes communicate by sending messages.
2. What Is Mutual Exclusion?

Mutual exclusion means:

Only one process can access a shared resource at a time.

Shared Resources
Printer
File
Database
Network connection
Critical Section

The part of a program where the shared resource is accessed.

Why Mutual Exclusion Is Needed

If two processes update the same resource simultaneously:

Data may become inconsistent.
Results may be incorrect.
3. Challenges in Distributed Systems

In a single computer, semaphores and mutexes can be used.

In distributed systems:

Processes run on different machines.
They communicate only through messages.

So specialized distributed algorithms are needed.

4. Token-Based Mutual Exclusion

In token-based algorithms, a special message called a token circulates in the system.

Rules:

Only the process holding the token can enter the critical section.
After finishing, it passes the token to another process.

Because only one token exists, only one process can access the critical section at any time.

5. Token Ring Algorithm

Processes are arranged in a logical ring.

P1 → P2 → P3 → P4 → P1

The token moves around the ring continuously.

6. Working Principle
Initially, one process holds the token.
The token is passed to the next process.
If a process wants to enter the critical section, it uses the token.
After completing its work, it passes the token onward.
If it does not need the critical section, it immediately forwards the token.
7. Real-Life Analogy

Students sit in a circle with one pen.

Only the student holding the pen can write.
After writing, the pen is passed to the next student.

Mapping:

Real Life	Token Ring
Student	Process
Pen	Token
Writing	Critical Section
8. Example

Processes:

P1 → P2 → P3 → P4 → P1

Suppose P3 needs the critical section:

Token moves from P1 to P2 to P3.
P3 enters the critical section.
P3 exits and passes the token to P4.
9. Properties of Token Ring Algorithm
Safety

Only one token exists, so only one process can enter the critical section.

Fairness

Every process gets the token in order.

Starvation-Free

Each process eventually receives the token.

Deadlock-Free

The token keeps circulating.

10. Advantages
Simple to understand and implement.
Guaranteed mutual exclusion.
Fair access.
Low message overhead.
11. Disadvantages
If the token is lost, the system stops until a new token is created.
If a process crashes, the ring may break.
Waiting time increases as the number of processes grows.
12. Message Complexity

Each critical-section entry requires at most N - 1 token passes, where N is the number of processes.

13. Practical Implementation Overview
TokenRing.java
Simulates token passing in a single program.
Shows how data is sent after the sender gets the token.
Process.java
Uses sockets to create real processes.
Each process communicates with the next one.
14. Applications
Distributed databases
Shared printers
File servers
Cloud resource coordination
15. Conclusion

The Token Ring Algorithm is a token-based distributed mutual exclusion algorithm in which a single token circulates among processes arranged in a logical ring. A process can enter the critical section only when it possesses the token, ensuring mutual exclusion, fairness, and freedom from starvation.

PART 1: Code 1 – TokenRing.java
What This Program Does

This program simulates token passing and data transmission in a logical ring.

All nodes are represented by numbers (0, 1, 2, ...).
Token initially starts at node 0.
Sender receives the token.
Sender sends data.
Intermediate nodes forward the data.
Receiver receives the data.
Code 1
import java.util.*;

class TokenRing {
    public static void main(String args[]) {
        Scanner scan = new Scanner(System.in);

        System.out.print("Enter the number of nodes: ");
        int n = scan.nextInt();

        int token = 0;

        // Display ring
        System.out.println("\nLogical Ring:");
        for (int i = 0; i < n; i++) {
            System.out.print(i + " ");
        }
        System.out.println("0");

        // Input sender, receiver, and data
        System.out.print("\nEnter sender: ");
        int s = scan.nextInt();

        System.out.print("Enter receiver: ");
        int r = scan.nextInt();

        System.out.print("Enter data: ");
        int data = scan.nextInt();

        // Token passing
        System.out.print("\nToken passing: ");
        int i = token;
        while (i != s) {
            System.out.print(i + " -> ");
            i = (i + 1) % n;
        }
        System.out.println(s);

        // Sender sends data
        System.out.println("Sender " + s + " sending data: " + data);

        // Forwarding data
        i = (s + 1) % n;
        while (i != r) {
            System.out.println("Data " + data + " forwarded by " + i);
            i = (i + 1) % n;
        }

        // Receiver receives data
        System.out.println("Receiver " + r + " received data: " + data);

        scan.close();
    }
}

PART 2: Code 2 – Process.java
What This Program Does

This version creates multiple real Java processes that communicate using sockets.

Each process:

Opens a server socket.
Connects to the next process.
Waits for the token.
Enters the critical section.
Passes the token.
Code 2
import java.io.*;
import java.net.*;

public class Process {
    public static void main(String[] args) throws Exception {

        int id = Integer.parseInt(args[0]);
        int myPort = Integer.parseInt(args[1]);
        int nextPort = Integer.parseInt(args[2]);

        ServerSocket serverSocket = new ServerSocket(myPort);
        System.out.println("Process " + id + " listening on port " + myPort);

        Socket next = null;
        while (next == null) {
            try {
                next = new Socket("localhost", nextPort);
            } catch (Exception e) {
                Thread.sleep(1000);
            }
        }

        DataOutputStream out = new DataOutputStream(next.getOutputStream());

        Socket prev = serverSocket.accept();
        DataInputStream in = new DataInputStream(prev.getInputStream());

        if (id == 1) {
            Thread.sleep(2000);
            System.out.println("\nProcess 1 STARTS THE TOKEN\n");
            out.writeUTF("TOKEN");
        }

        while (true) {
            String msg = in.readUTF();

            if (msg.equals("TOKEN")) {
                System.out.println("Process " + id + " RECEIVED TOKEN");

                Thread.sleep(1000);
                System.out.println("Process " + id + " ENTERING CRITICAL SECTION");

                Thread.sleep(2000);
                System.out.println("Process " + id + " EXITING CRITICAL SECTION");

                Thread.sleep(2000);
                System.out.println("Process " + id + " PASSING TOKEN");

                out.writeUTF("TOKEN");
            }
        }
    }
}
