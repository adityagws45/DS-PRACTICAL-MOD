Practical Flow: Parallel Sum Using OpenMP
Aim
To compute the sum of an array in parallel using OpenMP and demonstrate how multiple threads work together.

1. Write the Program
Create a file named sum.c and write the code.
The program:


Stores 8 numbers in an array.


Creates multiple threads.


Divides the array among the threads.


Each thread computes a partial sum.


All partial sums are added to get the final total.



2. Compile the Program
Ubuntu
gcc -fopenmp sum.c -o sum
Windows (CMD or VS Code terminal)
gcc -fopenmp sum.c -o sum
This creates an executable file:


Ubuntu: sum


Windows: sum.exe



3. Set Number of Threads
export OMP_NUM_THREADS=4      # Ubuntu
set OMP_NUM_THREADS=4         # Windows
This tells OpenMP to create 4 threads.

4. Run the Program
Ubuntu

sum

Step 3: Divide the Work
Each thread gets:
chunk = n / num_threads = 8 / 4 = 2

Step 6: Update Shared Variable
Inside:

each thread safely adds its partial sum to total_sum.

Step 7: Print Final Result
Final Total Sum = 36

Why Thread 2 Printed Before Thread 1
Threads run concurrently, and the operating system schedules them independently. The print order is not guaranteed.
Any order is valid.
