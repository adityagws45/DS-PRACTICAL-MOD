BullyProcess.java – Output Explanation

BullyProcess.java is a socket-based implementation of the Bully Algorithm.

Each terminal window represents one independent process. The process with the highest active ID becomes the leader.

1. Example Commands Used

Open five terminals and run:

Terminal 1
java BullyProcess 0
Terminal 2
java BullyProcess 1
Terminal 3
java BullyProcess 2
Terminal 4

Terminal 5
java BullyProcess 4

4. Line-by-Line Explanation

Example:

Process X starts election

The process begins the Bully Algorithm.

It sends ELECTION messages to all higher-ID processes.

Process X received ELECTION

A lower-ID process contacted this process.

This process:

Sends back OK
Starts its own election
Process X received OK

A higher-ID process is alive.

So this process knows it cannot become leader.

Process X becomes LEADER

No higher-ID process responded.

Therefore, this process is the highest active process.

Process X accepts leader: 4

The process receives:

5. Detailed Election Flow

Suppose Process 0 starts the election.

To:

Process 1
Process 2

Step 2: Higher Processes Reply

Each higher process replies OK.

Step 3: Higher Processes Start Their Own Elections

Eventually Process 4 starts election.

Step 4: Process 4 Has No Higher Process

So it receives no OK.

Step 5: Process 4 Becomes Leader
Process 4 becomes LEADER
Step 6: Leader Announcement

Process 4 sends:

to all other processes.

7. Why Process 4 Wins

Because Process 4 is the highest process ID.

No process has a higher ID than 4.

Therefore, no one can challenge it.

8. Final Result
Process ID	Status
0	Accepts leader 4
1	Accepts leader 4

9. Viva Explanation

Each process runs in a separate terminal and listens on a unique port. When an election starts, a process sends ELECTION messages to all higher-ID processes. If any higher process responds with OK, the current process waits. The highest active process receives no OK, becomes the leader, and broadcasts a COORDINATOR message to all other processes.

10. One-Line Summary

In BullyProcess.java, all processes start elections, and the highest active process (Process 4) becomes the leader and informs all others.

RingProcess.java – Output Explanation

RingProcess.java is a socket-based implementation of the Ring Election Algorithm.
Each terminal represents one process in a logical ring.

In your execution, you started:

Process 1
Process 2
Process 3

The ring formed was:

1 → 2 → 3 → 1
1. Commands Used
Terminal 1
java RingProcess 1 5001 5002
Terminal 2
java RingProcess 2 5002 5003
Terminal 3
java RingProcess 3 5003 5001
2. Your Output
Terminal 1 (Process 1)
Process 1 starts election
Process 1 received ELECTION [1, 2, 3]
Process 1 elected Leader: 3
Terminal 2 (Process 2)
Process 2 received ELECTION [1, 2]
Process 2 knows Leader is: 3
Terminal 3 (Process 3)
Process 3 received ELECTION [1, 2, 3]

3. Step-by-Step Output Explanation
Terminal 1 – Process 1
Process 1 starts election

Process 1 is the initiator because the code contains:

if (id == 1)

So Process 1 sends:

ELECTION:1

to Process 2.

Process 1 received ELECTION [1, 2, 3]

The election message completed one full round and returned to Process 1.

The message now contains all participating process IDs:

[1, 2, 3]
Process 1 elected Leader: 3

Process 1 computes the maximum ID:

max(1, 2, 3) = 3

Therefore, Process 3 is selected as leader.

Terminal 2 – Process 2
Process 2 received ELECTION [1, 2]

Process 2 received ELECTION:1, added its own ID, and forwarded:

ELECTION:1,2

to Process 3.

Process 2 knows Leader is: 3

Later, Process 2 received:

LEADER:3

So it recognizes Process 3 as the coordinator.

Terminal 3 – Process 3
Process 3 received ELECTION [1, 2, 3]

Process 3 received ELECTION:1,2, added its own ID, and forwarded:

ELECTION:1,2,3

to Process 1.

Process 3 knows Leader is: 3

Process 3 receives the leader announcement and confirms that it is the leader.

4. Election Message Flow

1 → 2 → 3 → 1

[1]
[1,2]
[1,2,3]

Highest ID = 3
5. Leader Announcement Flow
LEADER:3

1 → 2 → 3 → 1

All processes learn that Process 3 is the coordinator.

6. Why Process 3 Became Leader

The Ring Algorithm selects the process with the highest ID.

Among:

1, 2, 3

the highest is:

3

So Process 3 becomes the leader.

7. Final Result
Process	Role
Process 1	Initiator
Process 2	Participant
Process 3	Leader
8. Viva Explanation

Process 1 initiated the election by sending an election message containing its ID. Process 2 added its ID and forwarded the message to Process 3. Process 3 added its ID and forwarded the message back to Process 1. After receiving the list [1, 2, 3], Process 1 selected the highest ID, which was 3, as the leader. A leader announcement was then circulated, and all processes recognized Process 3 as the coordinator.
