Server.java
Client.java

Theory of CORBA Calculator Practical

Before writing code, it is important to understand the theory behind this practical.

1. What is CORBA?

CORBA (Common Object Request Broker Architecture) is a technology that allows a program running on one machine to call methods of an object running on another machine.

In simple words:

CORBA enables communication between distributed applications as if they were local.

2. What is a Distributed System?

A distributed system consists of multiple computers or processes working together.

Example
Client program runs on one computer.
Server program runs on another computer.
Client requests a service.
Server processes the request and returns the result.

In this practical:

Client sends two numbers.
Server performs calculation.
Result is sent back to client.
3. Real-Life Analogy

Think of ordering food in a restaurant.

Component	Real-Life Equivalent
Client	Customer
Server	Chef
Naming Service	Waiter
IDL	Menu Card
ORB	Communication System
Flow
Customer orders food.
Waiter finds the chef.
Chef prepares food.
Waiter delivers food.

Similarly:

Client requests operation.
Naming service locates server.
Server executes operation.
Result is returned.
4. Why CORBA is Needed?

Without CORBA, client and server communication becomes difficult.

CORBA provides:

Language independence
Platform independence
Object location transparency
Automatic communication handling
5. What is ORB?

ORB (Object Request Broker) is the heart of CORBA.

It acts as a messenger between client and server.

Responsibilities of ORB
Sends client requests to server.
Receives responses from server.
Hides networking details.
Simple Definition

ORB is middleware that manages communication between distributed objects.

6. What is IDL?

IDL (Interface Definition Language) defines the methods that the server will provide.

It is a contract between client and server.

In This Practical
interface Calculator {
    double add(in double a, in double b);
    double subtract(in double a, in double b);
    double multiply(in double a, in double b);
    double divide(in double a, in double b);
};
Meaning

The calculator provides four operations.

7. Why IDL is Used?

Both client and server use the same interface definition.

This ensures they agree on:

Method names
Parameters
Return types
8. What is idlj?

idlj is the Java IDL compiler.

It converts the IDL file into Java files.

Command
idlj -fall Calculator.idl
Generated Files
Calculator.java
CalculatorPOA.java
CalculatorHelper.java
CalculatorHolder.java
CalculatorOperations.java
_CalculatorStub.java
9. Stub and Skeleton
Stub

Client-side proxy.

The client calls methods on the stub as if it were the real object.

Skeleton

Server-side dispatcher.

Receives requests and calls actual methods.

10. What is POA?

POA (Portable Object Adapter) connects server objects to the ORB.

It activates objects and handles incoming requests.

11. What is Naming Service?

Naming service stores object references using names.

Example

Server registers object as:

"Calculator"

Client looks it up using:

resolve_str("Calculator")
Simple Definition

Naming service works like a phone directory.

12. What is tnameserv?

tnameserv starts the CORBA naming service.

Command
tnameserv -ORBInitialPort 1050

Port 1050 is used for communication.

13. Server Theory

The server performs these steps:

Initialize ORB.
Activate POA.
Create CalculatorImpl.
Convert servant to object reference.
Register object in naming service.
Wait for client requests.
14. Client Theory

The client performs these steps:

Initialize ORB.
Connect to naming service.
Look up "Calculator".
Get remote object reference.
Call methods.
Display results.
15. Complete Working Flow
Client enters 10 and 5
        ↓
Client calls add(10, 5)
        ↓
Stub packages request
        ↓
ORB sends request
        ↓
Server skeleton receives request
        ↓
CalculatorImpl.add() executes
        ↓
Result 15 returned
        ↓
Client displays 15
